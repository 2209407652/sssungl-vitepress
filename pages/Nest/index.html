介绍 #

Nest 是一个用于构建高效，可扩展的 Node.js 服务器端应用程序的框架。

常用命令 #

安装 #

使用 CLI 安装:

sh
npm i -g @nestjs/cli
nest new project-name

打开浏览器导航到 http://localhost:3000/

热重载 #

使用 CLI

1.首先需要安装一下:

sh
pnpm i --save-dev webpack-node-externals run-script-webpack-plugin webpack

2.配置 创建一个 webpack-hmr.config.js , 它是一个 webpack 的一个配置文件，并将其放入根目录。

js
const nodeExternals = require("webpack-node-externals");
const { RunScriptWebpackPlugin } = require("run-script-webpack-plugin");

module.exports = function (options, webpack) {
  return {
    ...options,
    entry: ["webpack/hot/poll?100", options.entry],
    externals: [
      nodeExternals({
        allowlist: ["webpack/hot/poll?100"],
      }),
    ],
    plugins: [
      ...options.plugins,
      new webpack.HotModuleReplacementPlugin(),
      new webpack.WatchIgnorePlugin({
        paths: [/\.js$/, /\.d\.ts$/],
      }),
      new RunScriptWebpackPlugin({ name: options.output.filename }),
    ],
  };
};

此函数获取包含默认 webpack 配置的原始对象，并返回一个已修改的对象和一个已应用的 HotModuleReplacementPlugin 插件。

3.热模块更换 为了启用 HMR，请打开应用程序入口文件（ main.ts ）并添加一些与 Webpack 相关的说明，如下所示：

ts
declare const module: any;

async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  await app.listen(3000);

  if (module.hot) {
    module.hot.accept();
    module.hot.dispose(() => app.close());
  }
}
bootstrap();

就这样。为了简化执行过程，请将这两行添加到 package.json 文件的脚本中。

json
"start:dev": "nest build --webpack --webpackPath webpack-hmr.config.js --watch"

运行npm run start:dev :

第一步 #

src 目录下的核心文件。

src
 ├── app.controller.spec.ts
 ├── app.controller.ts
 ├── app.module.ts
 ├── app.service.ts
 └── main.ts

核心文件的简要概述

filesummary
app.controller.ts带有单个路由的基本控制器示例。
app.controller.spec.ts对于基本控制器的单元测试样例
app.module.ts应用程序的根模块。
app.service.ts带有单个方法的基本服务
main.ts应用程序入口文件。它使用 NestFactory 用来创建 Nest 应用实例。

main.ts 包含一个异步函数，负责引导我们的应用程序

ts
declare const module: any;
import { NestFactory } from "@nestjs/core";
import { AppModule } from "./app.module";

async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  await app.listen(3000);

  if (module.hot) {
    module.hot.accept();
    module.hot.dispose(() => app.close());
  }
}
bootstrap();

控制器 #

控制器负责处理传入的请求和向客户端返回响应。

路由 #

使用 CLI 创建控制器 nest g controller cats。

生成两个文件。cats.controller.spec.ts、cats.controller.ts

ts
/* cats.controller.ts */
import { Controller, Get } from "@nestjs/common";

@Controller("cats")
export class CatsController {
  @Get()
  findAll(): string {
    return "This action return all cats";
  }
}

Get() HTTP 请求方法装饰器告诉 Nest 为 HTTP 请求的特定端点创建处理程序。 如 @Get() 为 GET /, 我们已经为每个 route(cats) 声明了一个前缀，并且没有在装饰器中添加任何路由信息, 因此 GET /cats 请求会映射到 findAll 处理程序。 例如: 装饰器 @Get('profile') 会为 GET /cats/profile。

Request #

当处理程序需要访问客户端的请求细节时。(默认为 express)

ts
/* cats.controller.ts */
import { Controller, Get, Req } from "@nestjs/common";
import { Request } from "express";

@Controller("cats")
export class CatsController {
  @Get()
  findAll(@Req() request: Request): string {
    console.log(request);

    return "This action return all cats";
  }
}

当请求/cats时查看打印出来的结果。

TIP

为了能够在express中使用TypeScript, 安装@types/express

装饰器和对象

装饰器对象
@Request()，@Req()req
@Response()，@Res()*res
@Next()next
@Session()req.session
@Param(key?: string)req.params/req.params[key]
@Body(key?: string)req.body/req.body[key]
@Query(key?: string)req.query/req.query[key]
@Headers(name?: string)req.headers/req.headers[name]
@Ip()req.ip
@HostParam()req.hosts

Nest 为所有标准 HTTP 方法提供了相应的装饰器@Post()、@Put()、@Delete()、@Patch()、@Options()以及@Head(), 此外，@All() 则用于定义一个用于处理所有 HTTP 请求方法的处理程序。

路由通配符 #

ts
@Get('ab*cd')
findAll() {
  return 'This route uses a wildcard';
}

路由路径 'ab*cd' 将匹配 abcd 、ab_cd 、abecd 等。

状态码 #

默认情况下，响应的状态码总是默认为 200，除了 POST 请求（默认响应状态码为 201）

ts
@Post()
@HttpCode(204)
create() {
  return 'This action adds a new cat';
}

Headers #

ts
@Post()
@Header('Cache-Control', 'none')
create() {
  return 'This action adds a new cat';
}

控制器和服务

ts
// cats.controller.ts
import { Controller, Get, Delete, Param } from "@nestjs/common";
import { CatsService } from "./cats.service";

@Controller("cats")
export class CatsController {
  constructor(private readonly catsService: CatsService) {}

  @Get()
  findAll() {
    return this.catsService.findAll();
  }

  @Get(":id")
  findById(@Param() params) {
    return this.catsService.findOne(params.id);
  }

  @Delete(":id")
  removeOne(@Param("id") id: string) {
    return this.catsService.remove(id);
  }
}

Create a Basic Service #

ts
// cats.service.ts

import { Injectable, NotFoundException } from "@nestjs/common";
import { Cats } from "./entities/cats.entity";

@Injectable()
export class CatsService {
  private cats: Cats[] = [
    {
      id: 1,
      name: "juanjuan",
      age: 2,
      varieties: ["none", "ace"],
    },
    {
      id: 2,
      name: "dajuan",
      age: 1,
      varieties: ["abc", "cba"],
    },
  ];

  findAll() {
    return this.cats;
  }

  findOne(id: string) {
    const cat = this.cats.find((item) => item.id === +id);
    if (!cat) {
      throw new NotFoundException(`Cats #id:${id} not found`);
    }
    return cat;
  }

  remove(id: string) {
    const catIndex = this.cats.findIndex((item) => item.id === +id);
    if (catIndex >= 0) {
      this.cats.splice(catIndex, 1);
    }
  }
}

DTO - Data Transfer Objects #

为数据传输对象定义形状

数据验证 - Validate #

nest 提供了 ValidationPipe 来对数据进行校验。 注册全局管道:

ts
// main.ts
import { ValidationPipe } from "@nestjs/common";

async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  app.useGlobalPipes(new ValidationPipe());
  await app.listen(3000);
}
bootstrap();

安装两个包

sh
pnpm i class-validator class-transformer

使用

ts
// create-cat.dto.ts
import { IsString } from "class-validator";

export class CreateCatDto {
  @IsString()
  readonly name: string;

  @IsString()
  readonly age: number;

  @IsString({ each: true }) // each: true 表示期望是一个字符串数组
  readonly varieties: string[];
}

当传递的数据不符合验证时，返回 400 状态码以及消息。

不符合验证示例

json
// post 添加数据
{
  "name": "wang"
}

// 响应示例
{
	"statusCode": 400,
	"message": [
		"age must be a string",
		"each value in varieties must be a string"
	],
	"error": "Bad Request"
}

安装类型转换包

sh
pnpm i @nestjs/mapped-types
ts
// update-cat.dto.ts
import { PartialType } from "@nestjs/mapped-types";
import { CreateCatDto } from "./create-cat.dto";

export class UpdateCatDto extends PartialType(CreateCatDto) {}

PartialType 自动返回参数的所有类型，并且添加@IsOption 全部变成可选参数。

DTO - Handing Malicious Request Data #

添加白名单可以自动过滤不符合验证的参数。

ts
// main.ts
app.useGlobalPipes(
  new ValidationPipe({
    whitelist: true,
  })
);

// ...

// cats.service.ts
addOne(cat) {
  this.cats.push(cat);
  return cat;
}

响应示例

json
// Body
{
  "name": "ss",
  "age": 20,
  "varieties": ["1", "2"],
  "other": false
}
// Response
{
	"name": "ss",
	"age": 20,
	"varieties": [
		"1",
		"2"
	]
}

可以看到自动过滤了不符合要求的数据(other)。

添加多余参数抛出错误

ts
// main.ts
app.useGlobalPipes(
  new ValidationPipe({
    whitelist: true,
    forbidNonWhitelisted: true,
  })
);

响应示例

json
// Body

    "age": 20,

Autotransform Payloads to DTO instances #

当负载符合类型的形状时，但实际上还不是类的实例 此时打印

ts
  @Post()
  addOne(@Body() createCatDto: CreateCatDto) {
    console.log(createCatDto instanceof CreateCatDto);

    return this.catsService.addOne(createCatDto);
  }

  // log
  false
  /* 全局启用自动转化 */
  // log
  true

全局启用自动转化:

ts
app.useGlobalPipes(
  new ValidationPipe({
    whitelist: true,
    transform: true,
    forbidNonWhitelisted: true,
  })
);

除此之外无论是params还是query都是通过网络字符串传递，所以接受的类型为string, 全局启用自动转化后

ts
  @Get(':id')
  findById(@Param('id') id: number) {
    console.log(typeof id);

    return this.catsService.findOne('' + id);
  }

  // log
  number

全局转化可能影响性能

Using mongodb #

1.安装

sh
npm install --save @nestjs/mongoose mongoose

2.将MongooseModule导入到AppModule中

ts
import { Module } from "@nestjs/common";
import { MongooseModule } from "@nestjs/mongoose";
import { AppController } from "./app.controller";
import { AppService } from "./app.service";
import { CatsModule } from "./cats/cats.module";

@Module({
  imports: [
    CatsModule,
    MongooseModule.forRoot(
      "mongodb://user:password@host:port/catsdemo?authSource=admin&readPreference=primary&appname=MongoDB%20Compass&directConnection=true&ssl=false"
    ),
  ],
  controllers: [AppController],
  providers: [AppService],
})
export class AppModule {}

3.使用装饰器创建模型

ts
// schemas/cats.schema.ts
import { Prop, Schema, SchemaFactory } from "@nestjs/mongoose";
import { Document } from "mongoose";

export type CatDocument = Cat & Document;

@Schema()
export class Cat extends Document {
  @Prop()
  name: string;

  @Prop()
  age: number;

  @Prop([String])
  varieties: string[];
}

export const CatSchema = SchemaFactory.createForClass(Cat);

4.再看看CatsModule

ts
import { Module } from "@nestjs/common";
import { MongooseModule } from "@nestjs/mongoose";
import { CatsController } from "./cats.controller";
import { CatsService } from "./cats.service";
import { Cat, CatSchema } from "./schemas/cats.schema";

@Module({
  imports: [MongooseModule.forFeature([{ name: Cat.name, schema: CatSchema }])],
  controllers: [CatsController],
  providers: [CatsService],
})
export class CatsModule {}

5.使用@InjectModel()装饰器将Cat模型注入到CatsService中

ts
import { Model } from "mongoose";
import { InjectModel } from "@nestjs/mongoose";
import { Injectable, NotFoundException } from "@nestjs/common";
import { Cat, CatDocument } from "./schemas/cats.schema";
import { CreateCatDto } from "./dto/create-cat.dto";

@Injectable()
export class CatsService {
  constructor(@InjectModel("Cat") private catModel: Model<CatDocument>) {}

  async findAll() {
    return this.catModel.find().exec();
  }

  async findOne(id: string) {
    const cat = this.catModel.findById(id);
    if (!cat) {
      throw new NotFoundException(`Cats #id:${id} not found`);
    }
    return cat;
  }

  async addOne(createCatDto: CreateCatDto) {
    const createdCat = new this.catModel(createCatDto);
    console.log(createdCat, "打印");

    return createdCat.save();
  }
}
